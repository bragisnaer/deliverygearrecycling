## Cross-Phase Integration Findings

Integration checker verified 47/49 exports are properly wired. Two **ship blockers** found:

### Ship Blocker 1: middleware.ts + proxy.ts coexistence

Both `apps/web/middleware.ts` and `apps/web/proxy.ts` exist. Next.js 16 requires only `proxy.ts`. Current state: `pnpm turbo build` fails. **Must resolve before production deploy.**

**Fix:** Remove `apps/web/middleware.ts` (the correct tenant resolution is already in `proxy.ts`).

### Ship Blocker 2: Migration journal missing entries

`packages/db/migrations/meta/_journal.json` does not include `0005_phase6_processing_dispatch_audit.sql` or `0006_financial_records.sql`. In a fresh deployment, Drizzle Kit will not apply these migrations, causing:
- Phase 6 audit triggers to not run (AUDIT-06 broken)
- Financial record auto-create trigger to not fire (FIN-01 broken)

**Fix:** Manually register both migrations in `_journal.json`.

- **CO2 avoided (ESG-04)**: `calculateCO2Avoided()` returns `formula_pending: true`. Acknowledged — awaiting formula agreement with reco/Wolt.
- **Pickup photo upload**: Photos collected but not sent to Supabase Storage. Planned gap; no requirement broken.
