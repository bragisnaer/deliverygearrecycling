## Ship Blocker — Production Database Deployment

**7 manual supplement SQL files must be applied before go-live**

These files are NOT in the Drizzle journal. `pnpm db:migrate` will never apply them. Running only Drizzle migrations against a blank production database produces a schema missing RLS grants, all trigger functions (audit trail, financial auto-create), notifications, and import schemas.

| File | What it installs | Affects |
|------|-----------------|---------|
| `0001_rls_and_triggers.sql` | `audit_log_trigger()` SECURITY DEFINER function, `enable row level security force`, all `GRANT` statements | AUDIT-01–06, ROUTE-04 |
| `0003_phase4_pickup_transport.sql` | Transport provider grants, pickup status enum | TRANS-01–10 |
| `0004_intake_trigger_rls.sql` | Intake RLS force, discrepancy trigger | INTAKE-01–08 |
| `0005_phase6_processing_dispatch_audit.sql` | `CREATE TRIGGER` on intake_records, processing_reports, outbound_dispatches | AUDIT-01, AUDIT-06 |
| `0006_financial_records.sql` | `financial_record_on_intake_insert` trigger (auto-creates financial record on intake) | FIN-01 |
| `0008_notifications_manuals.sql` | notifications, manual_pages, mute_preferences tables + RLS | NOTIF-01–04, MANUAL-01–04 |
| `0009_historical_import.sql` | import_jobs table + RLS | IMPORT-01–04 |

**Resolution:** `scripts/setup-local-db.sh` applies all 7 in the correct dependency order for local development. `LOCAL-BETA-CHECKLIST.md` (repo root) documents the production psql commands step-by-step.

**Verification query (AUDIT-06 specifically):**
```sql
SELECT trigger_name, event_object_table
FROM information_schema.triggers
WHERE trigger_name IN ('audit_intake_records','audit_processing_reports','audit_outbound_dispatches');
```
Expected: 3 rows.
