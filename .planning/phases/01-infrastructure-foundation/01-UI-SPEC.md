## Typography

All sizes are in `px` mapped to `rem` in Tailwind CSS v4 (`16px = 1rem` base).

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px | 400 (Regular) | 1.5 | Aileron | Form field values, table cell text, helper text |
| Label | 14px | 600 (Semibold) | 1.4 | Aileron | Form field labels, table column headers |
| Heading | 20px | 600 (Semibold) | 1.2 | Aileron | Page title ("System Settings"), section headings within tab |
| UI / Mono | 13px | 500 (Medium) | 1.4 | Consolas | Button labels, tab labels, badge text, link text |

**Two weights only:** 400 (Regular) and 600 (Semibold). Consolas 500 is acceptable within
the Consolas type role only — it does not expand the weight palette.

**Accent reserved for:**
- Primary save button fill (per-section Save buttons)
- Active tab underline indicator
- Inline edit pencil icon on hover
- Toast success icon
- Active state for toggle/switch controls

**Never use `#ED1C24` for:** body text, error states, headings, table cell text, or borders.

## Interaction Contracts

### Tab Navigation
- Tabs render client-side only; no page reload on tab switch
- Active tab: Reco Red (`#ED1C24`) bottom border (2px), Aileron 14px/600
- Inactive tab: `#000000` text, no border, opacity 0.6 on hover

### Save Buttons (per section)
- One primary "Save changes" button per Card section — not one global save
- Button state: default → loading (spinner, disabled) → success toast → default
- On success: `sonner` toast, bottom-right, message: "Settings saved." Duration: 3 seconds
- On error: `sonner` toast, bottom-right, message: "Failed to save. Please try again." Duration: 5 seconds, with retry affordance
- Button is disabled while its section's form is unchanged (pristine state)

### Inline Facility Table Editing
- Row cells display read-only by default
- Click any cell: activates inline text input for that cell, focus ring in `#ED1C24`
- Press Enter or click outside: commits change, row returns to read-only display
- Press Escape: cancels edit, restores previous value
- Changed rows show a subtle row-level indicator (left border in `#ED1C24`, 2px) until saved
- Table-level "Save changes" button appears (primary) when any row has unsaved edits
- New row (via "Add facility"): appends blank row at bottom, first cell auto-focused

### Archive Facility (Destructive Action)
- Archive icon button in the row action column (lucide `Archive` icon, `#9FA4A6`, 16px)
- Click: opens `dialog` component — does NOT immediately archive
- Dialog content:
  - Title: "Archive facility?"
  - Body: "Archiving [facility name] will remove it from the active facility list. This action can be reversed by reco-admin."
  - Buttons: "Cancel" (secondary), "Archive facility" (destructive variant — `#CC1018` fill)
- Archived facilities: row remains in table, Status badge changes to "Archived" (grey `#9FA4A6`)
- Archived rows are visually de-emphasised: text at 60% opacity

### Input Validation
- Exchange rate: numeric, minimum 1, maximum 999.99, two decimal places enforced
- Warehouse ageing threshold: integer, minimum 1, maximum 365
- Discrepancy threshold: integer, minimum 1, maximum 100 (percentage)
- Validation fires on blur and on submit
- Inline error: `#CC1018` helper text below input, Aileron 12px/400
