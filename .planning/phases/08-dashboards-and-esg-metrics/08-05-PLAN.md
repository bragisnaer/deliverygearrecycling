  <action>
    Create `apps/web/app/(client)/overview/actions.ts` with `'use server'` directive.

    These queries use `withRLSContext` — RLS enforces tenant isolation. The `locationId` parameter controls single-location vs cross-market scope.

    Import `withRLSContext` from wherever established (check auth-guard.ts or existing actions for the import path). Import `sql` from `drizzle-orm`. Import `sumMaterialWeights`, `calculateReuseRate` from `@/lib/esg-calculator`.

    **`getClientPickupActivity(claims: JWTClaims, locationId?: string)`:**
    Returns `{ active: { status: string; count: number }[], recent: { id: string; reference: string; status: string; created_at: string }[] }`.
    Query pickups table:
    - Active: `SELECT status, COUNT(*)::int AS count FROM pickups WHERE status NOT IN ('delivered', 'intake_registered', 'cancelled') GROUP BY status`
    - Recent: `SELECT id, reference, status, created_at FROM pickups ORDER BY created_at DESC LIMIT 5`
    - Add `AND location_id = ${locationId}` if locationId is provided (client role)
    - RLS handles tenant scoping

    **`getClientSentVsReceived(claims: JWTClaims, locationId?: string)`:**
    Returns `{ product_name: string; sent: number; received: number; discrepancy_pct: number; location_name?: string }[]`.
    Query joining pickup_lines (informed_quantity) and intake_lines (actual_quantity) via pickups.id = intake_records.pickup_id, grouped by product.
    - For client-global (no locationId): also group by location and include location_name
    - Discrepancy pct: `ROUND((ABS(SUM(actual) - SUM(informed)) * 100.0 / NULLIF(SUM(informed), 0))::numeric, 1)`
    - Filter last 90 days of deliveries

    **`getClientVolumeByQuarter(claims: JWTClaims, locationId?: string)`:**
    Returns `{ quarter: string; item_count: number; location_name?: string }[]`.
    Query intake_lines joined with intake_records:
    ```sql
    SELECT
      TO_CHAR(ir.delivery_date, 'YYYY-"Q"Q') AS quarter,
      SUM(il.actual_quantity)::int AS item_count
    FROM intake_lines il
    JOIN intake_records ir ON ir.id = il.intake_record_id
    WHERE ir.voided = false
    GROUP BY quarter
    ORDER BY quarter DESC
    LIMIT 8
    ```
    Add location_id filter for client role. Add location grouping for client-global.

    **`getClientEsgSummary(claims: JWTClaims, locationId?: string)`:**
    Returns `{ materials: MaterialWeightRow[]; reuseRate: number; totalItems: number }`.
    Use the same temporal composition join as ops ESG (08-RESEARCH.md Pattern 1) but scoped by RLS. RLS filters by tenant_id automatically. Add optional location_id filter.
    Pass results through `sumMaterialWeights()` and `calculateReuseRate()`.

    **`getClientLocations(claims: JWTClaims)`:**
    Returns `{ id: string; name: string; country: string }[]` for client-global drill-down.
    Query locations table — RLS scopes to tenant.

    All functions accept `claims: JWTClaims` as first parameter for `withRLSContext`. The page.tsx extracts claims from session.
