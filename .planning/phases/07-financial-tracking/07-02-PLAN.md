  <action>
    Replace the stub Server Actions in `apps/web/app/(ops)/financial/actions.ts` with full implementations. Keep ALL existing pure functions and auth helpers from Plan 01 unchanged. Add these Server Actions:

    1. `getFinancialRecords()`:
       - Call `requireFinancialAccess()` for auth
       - Use `withRLSContext(user, async (tx) => { ... })`
       - SELECT from financialRecords JOIN intakeRecords (for reference, delivery_date, voided) JOIN tenants (for tenant name)
       - Filter: `eq(intakeRecords.voided, false)`
       - Order by: `desc(financialRecords.created_at)`
       - Return array of objects with: id, intake_reference (intakeRecords.reference), tenant_name, delivery_date, transport_cost_eur, estimated_invoice_amount_eur, invoice_status, invoice_number, invoice_date, created_at

    2. `getFinancialRecord(id: string)`:
       - Call `requireFinancialAccess()` for auth
       - Use `withRLSContext`
       - SELECT financialRecords + intakeRecords (reference, delivery_date, pickup_id, tenant_id) + tenants (name)
       - WHERE `eq(financialRecords.id, id)`
       - Return single object or null

    3. `calculateAndStoreInvoiceAmount(intakeRecordId: string)`:
       - Call `requireRecoAdmin()` for auth (write operation)
       - Use `withRLSContext(user, async (tx) => { ... })`
       - Step 1: Get intake record (pickup_id, outbound_shipment_id)
       - Step 2: Get intake lines (product_id, actual_quantity)
       - Step 3: For each line, query productPricing WHERE product_id = line.product_id AND isNull(effective_to) — get price_eur
       - Step 4: If intakeRecord.pickup_id exists:
         - Query transportBookings WHERE pickup_id = intakeRecord.pickup_id — get transport_cost_market_to_destination_eur (leg 1)
         - Query outboundShipmentPickups WHERE pickup_id = intakeRecord.pickup_id — get allocated_cost_eur (leg 2)
       - Step 5: Call `assembleTransportCost(leg1, leg2)` for total transport cost
       - Step 6: Build lines array: `lines.map(l => ({ actual_quantity: l.actual_quantity, price_eur: pricingMap.get(l.product_id) ?? null }))`
       - Step 7: Call `calculateInvoiceAmount(linesWithPricing, transportCost)` for estimated amount
       - Step 8: UPDATE financialRecords SET transport_cost_eur, estimated_invoice_amount_eur, updated_at WHERE intake_record_id = intakeRecordId
       - Step 9: `revalidatePath('/financial')`
       - Return `{ transport_cost_eur, estimated_invoice_amount_eur, missing_pricing }` where missing_pricing is array of product_ids with no current pricing

    4. `updateInvoiceFields(id: string, data: { invoice_status: string, invoice_number: string | null, invoice_date: string | null, notes: string | null })`:
       - Call `requireRecoAdmin()` for auth
       - Zod validation schema: `invoice_status` must be one of 'not_invoiced', 'invoiced', 'paid'; `invoice_number` optional string; `invoice_date` optional ISO date string; `notes` optional string
       - Use `withRLSContext(user, async (tx) => { ... })`
       - UPDATE financialRecords SET invoice_status, invoice_number, invoice_date (parse from ISO string to Date if not null), notes, updated_at = new Date() WHERE id
       - `revalidatePath('/financial')`
       - Return `{ success: true }`

    5. `getUninvoicedAlerts(thresholdDays?: number)`:
       - Call `requireFinancialAccess()` for auth
       - Use `withRLSContext`
       - Read system_settings for warehouse_ageing_threshold_days (use as default if thresholdDays not passed)
       - Query: SELECT count(*), sum(estimated_invoice_amount_eur) FROM financialRecords JOIN intakeRecords WHERE invoice_status = 'not_invoiced' AND intakeRecords.voided = false AND intakeRecords.delivered_at < NOW() - INTERVAL threshold days
       - Also query monthly estimate: sum(estimated_invoice_amount_eur) WHERE invoice_status = 'not_invoiced' AND created_at >= start of current month
       - Return `{ overdue_count, overdue_total_eur, monthly_uninvoiced_eur }`
