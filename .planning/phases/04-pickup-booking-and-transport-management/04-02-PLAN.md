<objective>
Build the client-facing pickup booking form and Server Action. A client or client-global user can fill out the form with dynamic product quantities from their tenant's registry, pallet count, preferred date (72h minimum lead time enforced), notes, and up to 5 photos. On submit, the Server Action validates, inserts the pickup + lines via withRLSContext, and the DB trigger assigns the PU-YYYY-NNNN reference.

Purpose: This is the primary client-facing feature of Phase 4 — the pickup booking form that replaces the current email/sheet-based booking workflow.
Output: Pickup form page, Server Action, unit tests for 72h validation.
</objective>
