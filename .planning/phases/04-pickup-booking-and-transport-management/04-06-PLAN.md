3. `requireRecoAdminOrTransport()` helper — allows reco-admin and transport roles
4. `getWarehouseInventory()` Server Action:
   - Fetch pickups WHERE status='at_warehouse', joined with:
     - locations (for market/address)
     - transport_bookings (for provider and arrival info)
     - transport_providers (for warehouse name)
     - pickup_lines joined with products (for product details)
   - For each pickup, compute `days_held = Math.floor((Date.now() - pickup.updated_at.getTime()) / (1000 * 60 * 60 * 24))` — updated_at is set when status changes to at_warehouse
   - Fetch `warehouse_ageing_threshold_days` from systemSettings
   - Return `{ pickups: [...], threshold }`

5. `checkAndCreateAgeingAlerts()`:
   - Called server-side when warehouse inventory page loads
   - For each pickup where days_held > threshold:
     - Check if an unread `warehouse_ageing_alert` notification already exists for this pickup (Pitfall 7: prevent duplicates)
     - If not, INSERT notification row: `type='warehouse_ageing_alert', title='Warehouse ageing alert: {reference}', body='Pickup {reference} has been at {warehouse} for {days_held} days (threshold: {threshold})', entity_type='pickup', entity_id=pickupId, user_id=null (global reco-admin alert)`
   - Per research Pitfall 7: "Before inserting a new ageing alert, check whether an unread ageing notification for that pickup already exists"

6. `updatePickupToAtWarehouse(pickupId: string)`:
   - Validate pickup has a consolidation transport booking
   - withRLSContext: UPDATE pickups SET status='at_warehouse', updated_at=now()
   - revalidatePath('/transport/outbound')
