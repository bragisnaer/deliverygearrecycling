### Task 2 — Determine correct apply order

The correct dependency order for a fresh PostgreSQL database is:

| Step | File | Type | Applies after |
|------|------|------|---------------|
| 1 | `0000_naive_mentallo.sql` | Drizzle | — (base schema) |
| 2 | `0001_curved_zaran.sql` | Drizzle | step 1 |
| 3 | `0001_rls_and_triggers.sql` | Manual | step 2 (adds FORCE RLS + GRANTs on tables from curved_zaran) |
| 4 | `0002_natural_exiles.sql` | Drizzle | step 3 |
| 5 | `0003_worthless_quicksilver.sql` | Drizzle | step 4 |
| 6 | `0003_phase4_pickup_transport.sql` | Manual | step 5 (adds pickup transport triggers on tables from worthless_quicksilver) |
| 7 | `0004_tense_chronomancer.sql` | Drizzle | step 6 |
| 8 | `0004_intake_trigger_rls.sql` | Manual | step 7 (adds intake trigger + RLS on tables from tense_chronomancer) |
| 9 | `0005_phase6_processing_dispatch_audit.sql` | Manual | step 8 (processing, dispatch, audit_log trigger) |
| 10 | `0006_financial_records.sql` | Manual | step 9 (financial_records table) |
| 11 | `0007_esg_dashboard_indexes.sql` | Drizzle (in journal) | step 10 (indexes — already applied by `db:migrate`) |
| 12 | `0008_notifications_manuals.sql` | Manual | step 11 (notifications table) |
| 13 | `0009_historical_import.sql` | Manual | step 12 (is_imported columns, import_jobs table) |

**Key insight:** `pnpm --filter @repo/db db:migrate` runs Drizzle Kit, which applies steps 1, 2, 4, 5, 7, 11 in journal order. The manual supplement files (steps 3, 6, 8, 9, 10, 12, 13) must be applied with `psql` interleaved at the correct points. Because `db:migrate` applies ALL journal entries at once, the manual files must be applied after `db:migrate` completes. This works because all manual supplements use `IF NOT EXISTS` or `IF NOT ALREADY EXISTS` guards and reference tables that exist after the full Drizzle migration run.

- [ ] Confirm every manual supplement SQL file uses `IF NOT EXISTS` / `CREATE OR REPLACE` / `ADD COLUMN IF NOT EXISTS` so they are safe to apply after Drizzle runs all its migrations in one pass
- [ ] Confirm that `0001_rls_and_triggers.sql` does not reference tables introduced after `0001_curved_zaran` (i.e., it only touches tables that exist after step 2)
- [ ] Confirm that `0003_phase4_pickup_transport.sql` does not reference tables introduced after `0003_worthless_quicksilver`
- [ ] Confirm that `0004_intake_trigger_rls.sql` does not reference tables introduced after `0004_tense_chronomancer`

### Task 3 — Create `scripts/setup-local-db.sh`

- [ ] Create the `scripts/` directory at the project root (it does not currently exist)
- [ ] Create `scripts/setup-local-db.sh` with the following content:

```bash
#!/usr/bin/env bash
# setup-local-db.sh
# Builds the complete database schema from scratch on a fresh PostgreSQL instance.
# Requires: DATABASE_URL env var pointing to an empty PostgreSQL database.
# Usage: bash scripts/setup-local-db.sh

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
MIGRATIONS_DIR="$SCRIPT_DIR/../packages/db/migrations"

if [ -z "${DATABASE_URL:-}" ]; then
  echo "ERROR: DATABASE_URL is not set." >&2
  echo "Export it before running: export DATABASE_URL=postgresql://user:pass@host:5432/dbname" >&2
  exit 1
fi

echo "==> Step 1: Running Drizzle journal migrations (pnpm db:migrate)..."
pnpm --filter @repo/db db:migrate

echo "==> Step 2: Applying manual supplement SQL files in dependency order..."

apply() {
  local file="$MIGRATIONS_DIR/$1"
  echo "    Applying: $1"
  psql "$DATABASE_URL" -f "$file"
}

apply "0001_rls_and_triggers.sql"
apply "0003_phase4_pickup_transport.sql"
apply "0004_intake_trigger_rls.sql"
apply "0005_phase6_processing_dispatch_audit.sql"
apply "0006_financial_records.sql"
# 0007_esg_dashboard_indexes.sql is already applied by db:migrate (journal idx 5) — skip
apply "0008_notifications_manuals.sql"
apply "0009_historical_import.sql"

echo "==> Step 3: Running Wolt seed data..."
pnpm --filter @repo/db seed:wolt

echo ""
echo "Database setup complete."
echo "Tables, triggers, RLS policies, GRANTs, and indexes are all applied."
echo "Wolt seed data is present."
```

- [ ] Make the script executable: `chmod +x scripts/setup-local-db.sh`
- [ ] Add a comment block at the top of the script explaining the Drizzle-vs-manual two-class migration architecture so future maintainers understand why manual files exist outside the journal
