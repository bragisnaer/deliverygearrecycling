This is a Server Component (no 'use client' directive needed — react-markdown supports SSR).

```typescript
import ReactMarkdown from 'react-markdown'
import remarkGfm from 'remark-gfm'
import rehypeRaw from 'rehype-raw'

export function ManualRenderer({ content }: { content: string }) {
  return (
    <div className="manual-content">
      <ReactMarkdown
        remarkPlugins={[remarkGfm]}
        rehypePlugins={[rehypeRaw]}
      >
        {content}
      </ReactMarkdown>
    </div>
  )
}
```

**1. Create `apps/web/app/(client)/manual/actions.ts`:**

`'use server'` directive.

Import `auth` from `@/auth`, `db`, `manualPages`, `withRLSContext` from `@repo/db`, `eq`, `and`, `asc` from `drizzle-orm`.

**`getClientManualPages()`:**
- Get session via `auth()`, throw if not authenticated
- Build user object with sub
- Use `withRLSContext(user, async (tx) => { ... })` to SELECT from `manualPages` WHERE `context = 'client'` AND `published = true` ORDER BY `display_order ASC, title ASC`
- NOTE: RLS already filters to context='client' AND published=true for client_role, but explicit WHERE is defence-in-depth
- Return array of `{ id, slug, title, display_order }`

**`getClientManualPage(slug: string)`:**
- Same auth pattern
- `withRLSContext` SELECT from `manualPages` WHERE `context = 'client'` AND `slug = slug` AND `published = true`, limit 1
- Return full page object or null
