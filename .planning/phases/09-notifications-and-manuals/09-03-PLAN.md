**1. Create `apps/web/app/(ops)/notifications/actions.ts`:**

```typescript
'use server'
```

Import `auth` from `@/auth`, `db`, `notifications`, `notificationMutePreferences`, `withRLSContext` from `@repo/db`, `eq`, `and`, `desc` from `drizzle-orm`, `isCritical` from `@/lib/notifications`.

**`getUnreadCount()`:**
- Get session via `auth()`, throw if no session
- Build user object: `{ ...session.user, sub: session.user.id! }`
- Use `withRLSContext(user, async (tx) => { ... })` to count notifications where `read = false` and `user_id = user.sub`
- Return count as number

**`getRecentNotifications(limit = 10)`:**
- Same auth pattern
- Use `withRLSContext` to SELECT from notifications, ordered by `created_at` DESC, limit parameter
- Return array of notification rows

**`getNotifications(page = 1, pageSize = 20)`:**
- Same pattern, with OFFSET = (page - 1) * pageSize
- Return `{ items, total }` (total via separate count query)

**`markNotificationRead(notificationId: string)`:**
- Same auth pattern
- `withRLSContext` UPDATE notifications SET read = true WHERE id = notificationId
- Call `revalidatePath('/notifications')`

**`markAllRead()`:**
- Same auth pattern
- `withRLSContext` UPDATE notifications SET read = true WHERE user_id = user.sub AND read = false
- Call `revalidatePath('/notifications')`

**`saveMutePreference(notificationType: string, muted: boolean)`:**
- Same auth pattern
- **Guard:** If `isCritical(notificationType)`, throw new Error('Critical notification types cannot be muted')
- Use `withRLSContext` to upsert into `notificationMutePreferences`:
  - Try INSERT with `onConflictDoUpdate` on (user_id, notification_type) setting muted value
- Call `revalidatePath('/notifications')`

**`getMutePreferences()`:**
- Same auth pattern
- SELECT all from `notificationMutePreferences` where `user_id = user.sub`
- Return array of `{ notification_type, muted }` objects
