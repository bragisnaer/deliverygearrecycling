     ```typescript
       total_rows: rows.length,
       valid_rows: valid.length,
       error_count: allErrors.length,
       rows_json: JSON.stringify(valid),
       errors_json: JSON.stringify(allErrors),
       column_mapping_json: columnMapping || null,
       created_by: session.user.id,
     }).returning()
     ```

b) Export `async function commitImport(jobId: string)`:
   - Auth check: `const session = await auth()` — verify reco-admin
   - Load job from import_jobs by id using raw db
   - If job not found or job.status !== 'ready': throw Error('Job not found or not ready for commit')
   - If job.status === 'committed': throw Error('Job already committed')
   - Parse rows from job.rows_json
   - Import `pickupLines` from `@repo/db/schema` (for pickup_log line items)
   - Import `intakeLines` from `@repo/db/schema` (for intake_log line items)
   - Determine target table from job.source using a switch statement:

     **pickup_log:**
     ```typescript
     await withRLSContext({ sub: session.user.id, role: 'reco-admin' }, async (tx) => {
       for (const row of rows) {
         const [pickup] = await tx.insert(pickups).values({
           tenant_id: job.target_tenant_id,
           location_id: row.location_id,
           pallet_count: row.pallet_count,
           preferred_date: new Date(row.preferred_date),
           status: row.status || 'delivered',
           notes: row.notes || null,
           submitted_by: session.user.id,
           is_imported: true,
         }).onConflictDoNothing().returning({ id: pickups.id })
         // Insert pickup lines if present
         if (row.lines && Array.isArray(row.lines)) {
           await tx.insert(pickupLines).values(row.lines.map((l: { product_id: string; quantity: number }) => ({
             pickup_id: pickup.id,
             product_id: l.product_id,
             quantity: l.quantity,
           })))
         }
       }
     })
     ```

     **intake_log:**
     Similar pattern — insert into intakeRecords with is_imported: true, is_unexpected: false, staff_name from row, delivery_date, prison_facility_id, tenant_id from job. Use `.onConflictDoNothing()` on the intakeRecords insert to prevent duplicates if the same source is imported twice. Then insert intakeLines per row (intake_record_id, product_id, actual_quantity). The IN-YYYY-NNNN trigger fires automatically. The create_financial_record trigger also fires automatically.

     **greenloop:**
     Insert into processingReports with is_imported: true, activity_type, staff_name, product_id, report_date, prison_facility_id, tenant_id from job.

     **invoice_binder:**
     UPDATE existing financial_records (not insert — the trigger already created them from intake import). Use `db.update(financialRecords).set({ invoice_status, invoice_number, invoice_date, notes, updated_at: new Date() }).where(eq(financialRecords.intake_record_id, row.intake_record_id))`. Use withRLSContext for reco-admin.

     **transport_costs:**
     Attempt UPDATE first on existing transport_bookings by pickup_id. If no existing booking, INSERT with is_imported: true. Use `onConflictDoNothing()` for safety.

     ```typescript
     await db.update(importJobs).set({
       status: 'committed',
       committed_at: new Date(),
       updated_at: new Date(),
     }).where(eq(importJobs.id, jobId))
     ```
   - `revalidatePath('/import')`
   - Return `{ success: true, importedCount: rows.length }`
