Create `scripts/deploy-db-production.sh` modelled on `setup-local-db.sh` with these differences:

1. **Header comment block**: Same two-class migration architecture documentation as setup-local-db.sh, but title is "deploy-db-production.sh" and purpose states "Applies the complete database schema to a PRODUCTION Supabase project."

2. **Shebang and safety**: `#!/usr/bin/env bash` followed by `set -euo pipefail`

3. **Path resolution**:
```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
MIGRATIONS_DIR="$SCRIPT_DIR/../packages/db/migrations"
```

4. **DATABASE_URL guard**: Same as setup-local-db.sh — check `${DATABASE_URL:-}` is non-empty, print error to stderr and exit 1 if missing.

5. **Production safety prompt**: Before running anything, print a warning and require explicit confirmation:
```bash
echo "WARNING: This will apply ALL migrations to the database at:"
echo "  $DATABASE_URL"
echo ""
read -p "Continue? (y/N) " confirm
if [[ "$confirm" != "y" && "$confirm" != "Y" ]]; then
  echo "Aborted."
  exit 0
fi
```

6. **Step 1 — Drizzle migrations**: Same as setup-local-db.sh:
```bash
echo "==> Step 1: Running Drizzle journal migrations (pnpm db:migrate)..."
pnpm --filter @repo/db db:migrate
```

7. **Step 2 — Manual supplements**: Same `apply()` helper function and same 7 files in the same dependency order as setup-local-db.sh:
```bash
echo "==> Step 2: Applying manual supplement SQL files in dependency order..."

apply() {
  local file="$MIGRATIONS_DIR/$1"
  echo "    Applying: $1"
  psql "$DATABASE_URL" -f "$file"
}

apply "0001_rls_and_triggers.sql"
apply "0003_phase4_pickup_transport.sql"
apply "0004_intake_trigger_rls.sql"
apply "0005_phase6_processing_dispatch_audit.sql"
apply "0006_financial_records.sql"
# 0007_esg_dashboard_indexes.sql is already applied by db:migrate (journal idx 5) — skip
apply "0008_notifications_manuals.sql"
apply "0009_historical_import.sql"
```

8. **Step 3 — Trigger verification**: Unlike setup-local-db.sh (which runs seed), the production script verifies audit triggers exist:
```bash
echo "==> Step 3: Verifying audit triggers..."
TRIGGER_COUNT=$(psql "$DATABASE_URL" -t -A -c "SELECT count(*) FROM information_schema.triggers WHERE trigger_name IN ('audit_intake_records','audit_processing_reports','audit_outbound_dispatches')")
if [ "$TRIGGER_COUNT" -eq 3 ]; then
  echo "    OK: All 3 audit triggers present."
else
  echo "    WARNING: Expected 3 audit triggers, found $TRIGGER_COUNT." >&2
  echo "    Run the following to diagnose:" >&2
  echo "    psql \$DATABASE_URL -c \"SELECT trigger_name FROM information_schema.triggers WHERE trigger_name LIKE 'audit_%'\"" >&2
  exit 1
fi
```

9. **NO seed step**: Production script must NOT include `pnpm --filter @repo/db seed:wolt` or any seed command.

10. **Completion message**:
```bash
echo ""
echo "Production database deployment complete."
echo "Tables, triggers, RLS policies, GRANTs, and indexes are all applied."
echo "Audit triggers verified: intake_records, processing_reports, outbound_dispatches."
```

Then update the root `package.json` to add `"db:deploy": "bash scripts/deploy-db-production.sh"` to the `scripts` object, after the existing `"format"` entry.
