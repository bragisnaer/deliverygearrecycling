## What Was Built

### Schema: packages/db/src/schema/products.ts

Three Drizzle tables with full RLS following the established fail-closed pattern from auth.ts and tenants.ts:

**`products`** — Core product registry table
- `tenant_id` TEXT FK → tenants.id (cascade delete)
- `product_code` TEXT — unique per tenant enforced by DB partial index
- `product_group` TEXT nullable — groups product versions (e.g. "Bike Bag")
- `processing_stream` ENUM('recycling', 'reuse') NOT NULL
- `weight_grams` NUMERIC(10,2) nullable — clothing has unknown weight
- RLS: restrictive deny-all + reco-admin full CRUD + reco SELECT all + client SELECT WHERE tenant_id matches JWT

**`product_materials`** — Effective-dated material composition
- `product_id` UUID FK → products.id (cascade delete)
- `material_library_id` UUID FK → material_library.id (no cascade)
- `weight_grams` NUMERIC(10,2) NOT NULL
- `recycling_cost_per_kg_eur/dkk` NUMERIC(10,4) nullable
- `recycling_outcome` ENUM nullable
- `disassembly_photo_paths` TEXT[] nullable — up to 2 storage paths
- `effective_from` / `effective_to` — null effective_to means current record
- RLS: restrictive deny-all + reco-admin full CRUD + reco SELECT all + client SELECT via EXISTS subquery on parent product's tenant_id

**`product_pricing`** — Effective-dated pricing, reco-admin only
- `price_eur` / `price_dkk` NUMERIC(10,4) nullable (DKK for clothing, EUR for most)
- `effective_from` / `effective_to` dates
- RLS: restrictive deny-all + reco-admin full CRUD + reco SELECT — **no client_role policy** (fail-closed for sensitive master data)

### Migration: packages/db/migrations/0002_natural_exiles.sql

Generated by `drizzle-kit generate` then manually extended with:
- `CREATE UNIQUE INDEX products_code_tenant_uniq ON products (tenant_id, product_code)` — per-tenant product code uniqueness (PROD-01)
- `CREATE UNIQUE INDEX product_pricing_product_effective_from_uniq ON product_pricing (product_id, effective_from)` — prevents duplicate pricing rows for same date (Pitfall 4 from RESEARCH)
- `ALTER TABLE material_library FORCE ROW LEVEL SECURITY` — covers table created in 03-01
- `ALTER TABLE products FORCE ROW LEVEL SECURITY`
- `ALTER TABLE product_materials FORCE ROW LEVEL SECURITY`
- `ALTER TABLE product_pricing FORCE ROW LEVEL SECURITY`
