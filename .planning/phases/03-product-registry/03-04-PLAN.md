ADD the following Server Actions to the existing `apps/web/app/(ops)/products/actions.ts` file:

**getMaterials():**
- No auth guard (all authenticated users can read material_library)
- withRLSContext query: SELECT all from materialLibrary ORDER BY name
- Return `{ materials: Array<{id, name}> }`

**createMaterial(name: string):**
- requireRecoAdmin()
- Validate name with zod (string, min 1, max 100, trim)
- withRLSContext: INSERT into materialLibrary with name. Use `.onConflictDoNothing()` on name column.
- If conflict (already exists), SELECT the existing row by name to return its id
- Return `{ success: true, materialId: string }`

**getCurrentComposition(productId: string):**
- Validate productId with zod uuid
- withRLSContext query: SELECT from productMaterials INNER JOIN materialLibrary WHERE product_id = productId AND effective_to IS NULL, ORDER BY materialLibrary.name
- For each row with disassembly_photo_paths, generate signed URLs via `productsBucket.createSignedUrl()` with 1 hour expiry
- Return `{ composition: Array<{id, material_library_id, material_name, weight_grams, recycling_cost_per_kg_eur, recycling_cost_per_kg_dkk, recycling_outcome, disassembly_photo_urls}> }`

**saveMaterialComposition(productId: string, lines: Array<{material_library_id: string, weight_grams: string, recycling_cost_per_kg_eur?: string, recycling_cost_per_kg_dkk?: string, recycling_outcome?: string}>):**
- requireRecoAdmin()
- Validate all inputs with zod
- withRLSContext TRANSACTION:
  1. Close ALL current composition lines: UPDATE productMaterials SET effective_to = NOW(), updated_at = NOW() WHERE product_id = productId AND effective_to IS NULL
  2. INSERT new composition lines with effective_from = NOW(), effective_to = NULL for each line in the array
  3. Copy disassembly_photo_paths from old records to new records where the material_library_id matches (preserve photos across composition updates)
- revalidatePath(`/products/${productId}`)
- Return `{ success: true }`

**uploadMaterialPhoto(productId: string, materialLineId: string, formData: FormData):**
- requireRecoAdmin()
- Extract file, validate size <= 5MB, type startsWith 'image/'
- Check existing photo count on the material line (disassembly_photo_paths array length), reject if >= 2
- Upload to `getMaterialPhotoPath(tenantId, productId, materialLineId, filename)`
- UPDATE productMaterials: append path to disassembly_photo_paths array using `sql\`array_append(disassembly_photo_paths, ${path})\``
- revalidatePath(`/products/${productId}`)
- Return `{ success: true }`

**deleteMaterialPhoto(productId: string, materialLineId: string, photoPath: string):**
- requireRecoAdmin()
- Delete from storage: `productsBucket.remove([photoPath])`
- UPDATE productMaterials: remove path from array using `sql\`array_remove(disassembly_photo_paths, ${photoPath})\``
- revalidatePath(`/products/${productId}`)

**getPricingHistory(productId: string):**
- Validate productId with zod uuid
- withRLSContext query: SELECT from productPricing WHERE product_id = productId ORDER BY effective_from DESC
- Return `{ pricing: Array<{id, price_eur, price_dkk, effective_from, effective_to}> }`

**createPricingRecord(data: { product_id: string, price_eur?: string, price_dkk?: string, effective_from: string }):**
- requireRecoAdmin()
- Validate with zod: product_id (uuid), price_eur (string matching decimal pattern, optional), price_dkk (same, optional), effective_from (ISO date string). At least one of price_eur/price_dkk required.
- Parse effective_from as Date
- withRLSContext TRANSACTION:
  1. SELECT current record (WHERE product_id = data.product_id AND effective_to IS NULL)
  2. If current exists AND new effective_from <= current effective_from: return `{ error: 'New effective date must be after the current record effective date' }`
  3. Close current record: UPDATE SET effective_to = (new effective_from - 1 day), updated_at = NOW()
  4. INSERT new pricing record with effective_to = NULL
- revalidatePath(`/products/${data.product_id}`)
- Return `{ success: true }`
