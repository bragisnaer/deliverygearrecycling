| Secondary (30%) | `#F0EFE8` (--card / --secondary / --muted) | Cards (expected delivery cards, intake history rows), sidebar nav, tab backgrounds | globals.css |
| Accent (10%) | `#ED1C24` (--primary / --accent) | See reserved list below | globals.css |
| Destructive | `#CC1018` (--destructive) | Quarantine blocked state only | globals.css |
| Muted text | `#9FA4A6` (--muted-foreground) | Secondary labels, helper text, expected quantity display | globals.css |
| Border | `#E0DFD8` (--border) | Card borders, table row dividers, input borders | globals.css |

**Accent (`#ED1C24`) reserved for:**
1. Primary CTA button fill ("Registrer Levering", "Indsend Optælling")
2. Active tab indicator underline
3. "Register Another Delivery" / "Back to Home" post-submission buttons
4. Focus ring (`--ring`) on interactive elements
5. "Override and Allow" admin action button (ops portal quarantine queue)

**Accent is NOT used for:**
- Body text
- Warning/discrepancy indicators (these use amber/yellow — see Semantic Colors below)
- Informational badges or status pills (these use muted or border tokens)

**Semantic colors (inline, not in token set):**
- Discrepancy warning row highlight: `bg-amber-50 border-amber-200` (Tailwind utility — inline yellow, not a brand token)
- Discrepancy warning text badge: `text-amber-700`
- Quarantine blocked state banner: `bg-destructive/10 border-destructive text-destructive` (uses --destructive token)
- Success confirmation screen background: remains `--background`; success indicated by layout and copy, not a green colour (consistent with reco "no green clichés" rule)

## Navigation and Layout

### Prison Tablet Shell

```
/prison/
├── / (home)         — "Registrer Levering" primary action + History tab
├── /incoming        — Expected deliveries card grid
├── /intake/[id]     — Pre-populated intake form (expected flow)
├── /intake/new      — Manual intake form (unexpected flow)
└── /intake/[id]/success — Post-submit confirmation screen
```

- Shell uses full-viewport layout, no sidebar
- Single top bar: reco logo (left) + facility name (centre) + no user controls (facility-pre-authenticated, no logout needed on tablet)
- Navigation is linear: home → deliveries list → form → success → home (no back-navigation required except via "Tilbage til forsiden")

### Ops Portal Shell (extends existing)

```
/(ops)/intake/
├── /            — Intake queue (tabs: All / Discrepancy Flagged / Quarantine Blocked / Unexpected)
└── /discrepancy — Discrepancy analytics dashboard (tabs: By Country / By Product / By Facility)
```

- Uses existing ops sidebar nav (no new navigation shell needed)
- "Intake" added as a nav item in OpsNavBar

## Interaction Contracts

| Interaction | Behaviour |
|-------------|-----------|
| Quantity spinner: tap + | Increment count by 1; update diff badge immediately (client-side, no server round-trip) |
| Quantity spinner: tap − | Decrement count by 1; minimum 0; update diff badge immediately |
| Quantity spinner: direct keyboard input | Allowed on tablet; validates integer on blur; rejects non-numeric input |
| Expected delivery card: tap | Navigate to pre-populated intake form for that pickup; no intermediate confirm |
| Consolidated shipment card: tap expand | Expand inline to show nested per-pickup forms; does not navigate away |
| Discrepancy row highlight | Applied live as user types; amber highlight persists until value returns within threshold |
| Submit with discrepancy warnings | Submit proceeds; yellow summary banner shown above submit button; reco-admin notified server-side |
| Submit with quarantine block | Submit button disabled; destructive banner shown; no workaround on prison tablet — requires ops override |
| Override dialog: submit without reason | Confirm button disabled until textarea has ≥10 characters |
| Post-submit success screen | Auto-dismiss after 10 seconds to home, OR user taps one of the two explicit buttons |

## Accessibility

- All interactive elements meet WCAG AA 4.5:1 contrast (enforced by reco brand tokens already validated in Phase 2 WCAG gate)
- Quantity spinner: +/− buttons labelled with aria-label ("Forøg antal" / "Reducer antal") + aria-live region on the quantity display
- Discrepancy diff badge: role="status" so screen readers announce changes
- Quarantine block banner: role="alert" for immediate announcement
- All form inputs have visible labels (no placeholder-only patterns)
- Minimum touch target 44×44px enforced on all interactive elements in prison tablet shell (PROCESS-04, reco brand identity)
