    **Add to `apps/web/app/prison/actions.ts`:**

    `getExpectedDelivery(pickupId: string)`:
    - Fetch single pickup by ID with its lines, products, location, tenant
    - Return: `{ pickup_id, reference, client_name, tenant_id, origin_market, lines: [{ product_id, product_name, informed_quantity }] }`
    - Uses withRLSContext for facility isolation

    `submitIntake(formData: FormData)`:
    - Parse FormData: staff_name, delivery_date, origin_market, notes, pickup_id, lines[N][product_id], lines[N][actual_quantity], lines[N][batch_lot_number], lines[N][informed_quantity]
    - Validate with zod: staff_name required, delivery_date required, at least 1 line with actual_quantity > 0
    - Read discrepancy threshold from system_settings (default 15)
    - For each line: calculate discrepancy_pct via calculateDiscrepancyPct(actual, informed)
    - Set discrepancy_flagged on intake_record if ANY line exceeds threshold
    - Insert intake_record + intake_lines in a single withRLSContext transaction
    - If discrepancy_flagged: insert notification for reco-admin (type: 'discrepancy_detected', non-blocking try/catch)
    - Update pickup status to 'intake_registered' if pickup_id is present
    - Return { success: true, intakeId: record.id } or { error: string }
    - Note: quarantine logic handled in Plan 06 — for now, skip batch_flags check (will be added)
