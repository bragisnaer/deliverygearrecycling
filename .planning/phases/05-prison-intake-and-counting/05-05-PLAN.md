    **Add to `apps/web/app/prison/actions.ts`:**

    `getClientsForIntake()`:
    - Fetch all active tenants visible to the prison facility
    - Return: `{ id: string, name: string }[]`
    - Uses raw db (not withRLSContext) since prison_role RLS cannot read tenants cross-tenant (same pattern as Phase 4 admin email query)

    `getProductsForClient(tenantId: string)`:
    - Fetch active products for the given tenant
    - Return: `{ id: string, name: string, weight_grams: number }[]`
    - Uses withRLSContext

    `submitUnexpectedIntake(formData: FormData)`:
    - Parse FormData: staff_name, delivery_date, tenant_id (from client dropdown), origin_market, notes, lines[N][product_id], lines[N][actual_quantity], lines[N][batch_lot_number]
    - Validate: staff_name required, delivery_date required, tenant_id required, at least 1 line
    - Set `is_unexpected = true`, `pickup_id = null`, `informed_quantity = null` on all lines
    - No discrepancy detection (no informed_quantity to compare against)
    - Insert intake_record + intake_lines via withRLSContext
    - Insert notification for reco-admin: type='unexpected_intake', non-blocking try/catch
      - Use raw db for notification insert (prison_role cannot insert notifications for reco-admin cross-tenant — same as Phase 4 pattern)
    - Return { success: true, intakeId: record.id } or { error: string }
